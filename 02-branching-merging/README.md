# Part 2 — Branching & Merging

## 🎯 Goal
Get comfortable with the things that scare people most: a **merge conflict**, a **rebase**, and juggling **uncommitted work** when you need to switch branches.

## 🧠 What you practise here
- Merging a branch and resolving a conflict by hand
- Rebasing a feature branch to keep history tidy and linear
- Using `git stash` to park changes so you can switch branches safely

---

## 📝 The 3 scenarios

| # | File | What you practise |
|---|------|-------------------|
| 1 | `scenario-1-merge-conflict.md`  | resolving a merge conflict |
| 2 | `scenario-2-rebase-feature.md`  | rebasing a feature branch onto main |
| 3 | `scenario-3-stash-and-switch.md`| stashing work to switch branches |

Each scenario works on a ready-made repo under `sandbox/`. Run `python setup.py` from the lab root first (and any time you want to start over).

Step-by-step answers are in [`solutions/`](solutions). Try first, then check.
